Decidable Problems Concerning Regular Languages

Acceptance problem for DFAs: Check whether a given DFA accepts a given string or not.

This is same as testing whether ⟨B,w⟩\langle B, w \rangle⟨B,w⟩﻿ is a member of the language ADFAA_{DFA}ADFA​﻿. Showing that this language is decidable is same as showing the computational problem is decidable.

Theorem: ADFAA_{DFA}ADFA​﻿ is a decidable language.

Proof Idea:

Simulate DFA using a Turing machine.

Theorem: ANFAA_{NFA}ANFA​﻿ is a decidable language.

Proof Idea:

Convert NFA to DFA and simulate DFA on Turing machine.

Emptiness Testing for language of a finite automaton: Is the language empty?

Theorem: EDFAE_{DFA}EDFA​﻿ is a decidable language.

Proof Idea:

DFA is similar to a graph. Check if the ending state is reachable from the start state using some graph searching algorithm.

Theorem: Every context-free language is decidable.

Proof:

  1. Run TM created for proving ACFGA_{CFG}ACFG​﻿ on input ⟨G,w⟩\langle G,w \rangle⟨G,w⟩﻿.
  1. If this machine accepts, accept, else reject.

This theorem establishes a relationship between the four main classes of languages that have been described so far: regular, context-free, decidable, and Turing-recognizable.
