Formal definition of computation

A language is called a regular language if some finite automaton recognizes it.

Regular Operations

In theory of computation, objects are languages and we define operations on these languages.

We define three operations on languages, called regular operations and use them to study properties of regular languages.

Non-Determinism

So far in our discussion, every step of a computation follows in a unique way from the preceding step.

When we are at a step, we know what the next state will be — it is determined. We call this deterministic computation.

In a non-deterministic machine, several choices may exist for the next state at any point.

Nondeterminism is a generalization of determinism, so every deterministic finite automaton (DFA) is automatically a nondeterministic finite automaton (NFA).

Example of a NFA:

  1. NFAs can have multiple exiting transitions arrows for each symbol in the alphabet which is different from DFAs, where you can only have one exiting transition arrow .
  1. In DFA, labels on the transition arrows are symbols from the alphabet. NFAs can also include a special symbol ϵ\epsilonϵ﻿ which is not part of the alphabet.

Computation in NFA:

Suppose we are running an NDA on an input string and come to a state with multiple ways to proceed. After reading the symbol, the machine splits into multiple copies of itself and follows all the possibilities in parallel.

Each copy of the machine takes one of the possible ways to proceed and continues as before.

If there are subsequent choices, the machine keeps on splitting up.

Finally, if any one of these copies of the machine is an accept state, at end of the input, the NFA accepts the input string.

If a state with an ϵ\epsilonϵ﻿ is encountered, without reading any input, the machine splits and follows the ϵ\epsilon ϵ﻿ lines.

Nondeterminism can be viewed as a kind of parallel computation.

NFAs are useful as we will show later that every NFA can be converted to an equivalent DFA but usually building a NFA is much easier than building a DFA.

Lemma: If a language is regular, then it is described by a regular expression.

To prove this, we define a new type of finite automaton called generalized nondeterministic finite automaton, GNFA.

We first show DFA → GNFA and then GNFA → Regular expressions.

GNFAs are simply NFAs wherein transition arrows may have any regular expressions as labels instead of only members of the alphabet or ϵ\epsilonϵ﻿.

For convenience, we require that GNFAs always have a special form that meets the following conditions;

  1. The start state has arrows going out to every other state but no arrows coming in from any other state.
  1. There is only a single accept state, and it has arrows coming in from every other state but no arrows going out. Further, start state and end state are not same.
  1. Except for the start and accept states, one arrow goes from every state to every other state and also from each state to itself.

Converting DFA to GNFA is not hard and can be thought out.

Now we show how to convert a GNFA into a regular expression.
