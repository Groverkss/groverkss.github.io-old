There are many properties of a software that are usually more important than performance (Compatibility, Reliability, Portability, etc.)

Historically, software performance engineering was common because machine resources were limited. Programs would not "fit" without intense performance engineering.

Premature optimization is the root of all evil - Donald Knuth

Profile what part of the software is the bottleneck and apply optimizations on that part.

(About cpu performance not going up)

Vendor Solution: Multicore

To scale performance, processor manufacturers put many processing cores on the microprocessor chip.

Moore's Law continues to be obeyed with respect to computer performance.

But now the law is obeyed due to multicore processors as processor performance is not increasing considerably. These machines now have large number of cores, complex shared caches, GPU's etc.

Generally software must be adapted to utilize this hardware efficiently.

A modern multi core desktop processor contains parallel-processing cores, vector units, caches, perfetches, GPU's, hyperthreading, dynamic frequency scaling, etc.

How can we write software to utilize modern hardware efficiently?

Loop Order

The current order of loops is: i,j,ki, j, ki,j,k﻿

Changing the order to i,k,ji, k, ji,k,j﻿ will improve performance

Reason: spatial locality
