Constraint Satisfaction Problems

What is a search for?

Assumptions about the world: a single agent, deterministic actions, fully observed state, discrete state space.

We have used search for now:

Planning problems: sequences of actions

The path to the goal is the important thing

Paths have various costs, depths

Heuristics give problem-specific guidance: hey the goal is over this way mate

Another class of search problems:

Identification problems: assignments to variables

You just want the goal assignment, the path/cost doesn't matter

Generally paths at the same depth

CSPs are specialized class of identification problems

Constraint Satisfaction Problem

How are CSPs different from a standard search problems:

Standard Search Problems:

State is a "black box": arbitrary data structure

Goal test can be any function over states

Successor function can also be anything

Constraint Satisfaction Problems (CSPs):

A special subset of search problems

State is defined by variables XiX_iXi​﻿ with values from a domain DDD﻿: sometimes DDD﻿ depends on iii﻿

Goal test is a set of constraints specifying allowable combination of values for subsets of variables.

Because we have structure to the search, we can have better algorithms to solve.

Example: Map Coloring

Variables: WA, NT, Q, NSW, B, SA, T

Domains: D={red,green,blue}D = \{ red, green, blue\}D={red,green,blue}﻿

Constraints: Adjacent regions must have different colors

Implicit Constraints: WA ≠\neq​=﻿ NT

Explicit (WA, NT)∈{(red, green),(red, blue),… }(\text{WA, NT}) \in \{(red,\ green), (red,\ blue), \dots\}(WA, NT)∈{(red, green),(red, blue),…}﻿

Solutions: Assignments satisfying all constraints

Constraint Graphs

The constraint graph doesn't tell what the constraints are but tells where they are. An edge denotes that there is a constraint between them.

Binary CSP: Each constraint- related (at most) two variables.

Binary constraint graph: nodes are variables, arc show constraints

Varieties of CSPs

Finite Domains

Size ddd﻿ means O(dn)O(d^n)O(dn)﻿ complete assignments

Infinite domains (integers, strings, etc.)

Linear constraints solvable, nonlinear undecidable

Backtracking Search

Backtracking search is the basic uninformed algorithm

Idea 1: One variable at a time

Idea 2: Check constraints as you go

Depth-first search with these two improvements is called backtracking search

Algorithm:

Backtracking = DFS + variable-ordering + fail-on-violation

Improving Backtracking Search

Ordering

Variable Ordering: Minimum Remaining Values

Choose the variable with the fewest legal left values in its domain

"Fail-fast" Ordering

Variable Ordering: Least Constraining Value

Given a choice of variables, choose the least constraining value

i.e. the one that rules out the fewest values in the remaining variables
