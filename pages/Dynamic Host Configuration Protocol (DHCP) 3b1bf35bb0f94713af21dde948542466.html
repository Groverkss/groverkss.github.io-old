Dynamic Host Configuration Protocol (DHCP)

Communicating with IP

Need three things to communicate using IP (plus one)

  1. IP address: An address which can be given to other nodes so they can reply
  1. Subnet Mask: Needs to know which nodes are on its local network vs which nodes it needs to contact through a local gateway.
  1. Gateway Router: If the node is outside the network, the address of the first hop to communicate outside the network.
  1. A DNS server IP address is also useful but not necessary for IP.

DHCP

A machine can request a configuration from a DHCP server

  1. Movement: Just request a new configuration
  1. Configuration has a duration: a "lease", which can be renewed.
  1. Garbage Collection: Collect when lease expires.
Discover ---> Offer ---> Request ---> Ack ---> Release (optional)

To get a configuration, the client sends a Discover. The DHCP servers that hear that Discover, Offer a configuration. The client selects one of the Offers and sends a Request to the server. And then the server sends an Ack to acknowledge the request. The Release is optional and can be sent before the lease expires to release that configuration.

DHCP Sequence

Server A ---------------------------------------------------------------
         ^ Discover  
				 |
Client   ---------------------------------------------------------------
				 | Discover
			   v
Server B ---------------------------------------------------------------

The Client sends a discover message to all the available DHCP servers. Both Server A and Server B hear the message and both respond with offer.

Server A ---------------------------------------------------------------
         ^ Discover | 
				 |          v Offer A 
Client   ---------------------------------------------------------------
				 | Discover     ^ Offer B
			   v              | 
Server B ---------------------------------------------------------------

The client chooses Offer A and sends a Request to obtain it. The Offer B times out after a while.

Server A ---------------------------------------------------------------
         ^ Discover |         ^ Request |
				 |          v Offer A |         v Ack
Client   ---------------------------------------------------------------
				 | Discover     ^ Offer B
			   v              |                  Offer B Timeout
Server B ---------------------------------------------------------------

If the lease of offer A expires, it can request a configuration using the same pathway again. If it does not need the configuration anymore, the client can send a release to Server A.

Server A --------------------------------**************-----------------
         ^ Discover |         ^ Request |              ^ Release
				 |          v Offer A |         v Ack          |
Client   ---------------------------------------------------------------
				 | Discover     ^ Offer B
			   v              |                  Offer B Timeout
Server B ---------------------------------------------------------------

Communicating without IPv4

When the client has not configuration, it needs to communicate with the DHCP servers somehow.

  1. Ports used: UDP 67 (server) , 68 (client)
  1. Broadcast IP Address: 255.255.255.255. To obtain a configuration, the client sends a Discover message on the broadcast IP to obtain a configuration.
